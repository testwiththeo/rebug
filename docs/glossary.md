# Glossary - Rebug

### Session
A recorded sequence of DOM mutations, network requests, console logs, and user interactions captured during a single bug-hunting flow. Contains everything needed to replay and analyze the bug.

### Bug Marker
A manual flag inserted during recording when the QA discovers a bug. Marks the exact point in the timeline where the issue occurred.

### Session Package
The compressed, encrypted file containing all session events + metadata. Uploaded to storage and used for replay + analysis.

### Replay
A web-based playback of the recorded session showing DOM state changes synchronized with console and network logs at each point in time.

### Reproduction Steps
Numbered step-by-step instructions generated by AI from the session events that describe exactly how to replicate the bug.

### Root Cause Analysis
AI-generated diagnosis that identifies the most likely cause of the bug by correlating console errors, failed network requests, and DOM state.

### Duplicate Detection
Comparison of a new session against all existing bug reports using structural fingerprinting + LLM-based semantic similarity.

### Bug Report
A single filed bug containing: title, severity, steps, root cause, session replay link, environment info. May be linked to a Jira ticket.

### Impact Link
An automated connection between a previously rejected/ignored bug report and a later production incident that matches its symptoms.

### Sensitive Field Masking
Automatic detection and redaction of password inputs, credit card fields, and other PII during recording. Happens client-side before packaging.

### Session Fingerprint
A hash/vector representing the structural characteristics of a bug session (URLs interacted with, error patterns, DOM paths touched) used for duplicate detection.
